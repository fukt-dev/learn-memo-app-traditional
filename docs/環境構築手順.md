## 1. はじめに

### 1.1 この手順書について

本手順書は、開発未経験者がメモ帳アプリケーションを開発できるように、必要なソフトウェアのインストールと設定方法を **1つずつ丁寧に** 説明します。

### 1.2 環境構築で何をするのか

プログラムを書いて動かすには、以下のような「道具」が必要です：

| 道具 | 例え | 実際の役割 |
|------|------|-----------|
| **JDK** | 大工道具セット | Javaプログラムを書いて実行するための基本ツール |
| **IDE** | 作業台 | プログラムを書きやすくする編集ツール |
| **Maven** | 材料調達係 | 必要なライブラリを自動で集めてくれるツール |
| **PostgreSQL** | 倉庫 | データ（メモの内容など）を保存する場所 |
| **Docker** | コンテナ | PostgreSQLを簡単に起動・停止できる仕組み |
| **Git** | バージョン管理 | プログラムの変更履歴を記録するツール |

これらを1つずつインストールしていきます。

### 1.3 所要時間

- 初めての方: 約2-3時間
- 経験者: 約30分-1時間

## 3. Java Development Kit (JDK)

### 3.1 JDKとは何か

**JDK (Java Development Kit)** は、Javaプログラムを書いて実行するための基本ツールセットです。

**何ができるようになるか**:
- Javaのソースコード（`.java`ファイル）を書ける
- Javaプログラムをコンパイル（翻訳）して実行できる形（`.class`ファイル）にする
- 作成したプログラムを実行する

**なぜJDK 17が必要か**:
- Spring Boot 3.x は Java 17以上が必要
- 最新の機能とセキュリティアップデートを利用できる

### 3.2 JDKのインストール

#### 3.2.1 Windows

**方法1: Oracle JDK（推奨）**

1. **ダウンロード**
   - [Oracle JDK ダウンロードページ](https://www.oracle.com/java/technologies/downloads/) にアクセス
   - "Java 17" を選択
   - "Windows" タブから "x64 Installer" をダウンロード

2. **インストール**
   - ダウンロードした `.exe` ファイルをダブルクリック
   - インストーラーの指示に従う
   - インストール先は変更せず、デフォルトのまま進む（例: `C:\Program Files\Java\jdk-17`）

3. **環境変数の設定**

   環境変数とは、Windowsがプログラムを見つけるための「住所録」のようなものです。

   **手順**:
   ```
   1. スタートメニューを右クリック → 「システム」をクリック
   2. 「システムの詳細設定」をクリック
   3. 「環境変数」ボタンをクリック
   4. 「システム環境変数」の「新規」をクリック

   変数名: JAVA_HOME
   変数値: C:\Program Files\Java\jdk-17
   （実際のインストール先に合わせて調整）

   5. 「システム環境変数」から「Path」を選択して「編集」
   6. 「新規」をクリックして以下を追加:
      %JAVA_HOME%\bin

   7. すべてのウィンドウで「OK」をクリック
   ```

**方法2: Adoptium（無料版、推奨代替）**

1. [Adoptium](https://adoptium.net/) にアクセス
2. "Temurin 17 (LTS)" を選択
3. Windows用のインストーラーをダウンロード
4. インストーラーを実行（環境変数は自動設定されます）

#### 3.2.2 macOS

**Homebrew使用（推奨）**

```bash
# Homebrewがインストールされていない場合
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# JDK 17をインストール
brew install openjdk@17

# シンボリックリンク作成
sudo ln -sfn /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk

# 環境変数設定（~/.zshrc または ~/.bash_profile に追加）
echo 'export JAVA_HOME=/Library/Java/JavaVirtualMachines/openjdk-17.jdk/Contents/Home' >> ~/.zshrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc

# 設定を反映
source ~/.zshrc
```

#### 3.2.3 Linux (Ubuntu/Debian)

```bash
# パッケージリストを更新
sudo apt update

# JDK 17をインストール
sudo apt install openjdk-17-jdk -y

# 環境変数設定（~/.bashrc に追加）
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.bashrc

# 設定を反映
source ~/.bashrc
```

### 3.3 インストール確認

ターミナル（Windows: コマンドプロンプトまたはPowerShell、macOS/Linux: ターミナル）を開いて以下を実行：

```bash
# Javaのバージョン確認
java -version

# 期待される出力例:
# openjdk version "17.0.9" 2023-10-17
# OpenJDK Runtime Environment (build 17.0.9+9)
# OpenJDK 64-Bit Server VM (build 17.0.9+9, mixed mode, sharing)

# 環境変数の確認
echo $JAVA_HOME  # macOS/Linux
echo %JAVA_HOME%  # Windows (コマンドプロンプト)
$env:JAVA_HOME    # Windows (PowerShell)

# 期待される出力例:
# /Library/Java/JavaVirtualMachines/openjdk-17.jdk/Contents/Home (macOS)
# C:\Program Files\Java\jdk-17 (Windows)
```

**重要**: バージョンが `17.x.x` 以上であることを確認してください。

## 4. 統合開発環境 (IDE)

### 4.1 IDEとは何か

**IDE (Integrated Development Environment)** は、プログラムを書くための「超高機能なエディタ」です。

**IDEがないと困ること**:
- コードの入力補完がない → スペルミスが多発
- エラーの場所がわからない → デバッグに時間がかかる
- プロジェクトの管理が大変 → ファイルが増えると混乱

**IDEがあると便利なこと**:
- ✅ コードを途中まで書くと、続きを自動で提案してくれる
- ✅ エラーがあると赤線で教えてくれる
- ✅ クラスやメソッドをクリック1つで定義場所に飛べる
- ✅ デバッグ（バグ探し）が簡単にできる

### 4.2 IntelliJ IDEA のインストール

**IntelliJ IDEA** は、Java開発で最も人気のあるIDEです。

#### 4.2.1 Community Edition（無料版）

**すべてのOS共通手順**:

1. **ダウンロード**
   - [IntelliJ IDEA ダウンロードページ](https://www.jetbrains.com/idea/download/) にアクセス
   - **Community Edition（無料）** の「Download」をクリック
   - OSに合わせたインストーラーをダウンロード

2. **インストール**
   - Windows: `.exe` ファイルを実行
   - macOS: `.dmg` ファイルを開いてアプリケーションフォルダにドラッグ
   - Linux: `.tar.gz` を展開して `bin/idea.sh` を実行

3. **初回起動設定**
   ```
   1. IntelliJ IDEAを起動
   2. 「Do not import settings」を選択（初回のみ）
   3. テーマを選択（DarculaまたはLight）
   4. プラグインのインストールは後でOK
   5. 「Start using IntelliJ IDEA」をクリック
   ```

### 4.3 IDEの基本設定

#### 4.3.1 JDKの設定確認

IntelliJ IDEA起動後:

```
1. 「Configure」→「Project Structure」を開く
2. 「Project」タブを選択
3. 「Project SDK」でJDK 17が選択されているか確認
4. なければ「Add SDK」→「JDK」から先ほどインストールしたJDK 17を選択
```

#### 4.3.2 Maven設定確認（自動検出されているはず）

```
1. 「Configure」→「Settings」（macOS: Preferences）
2. 「Build, Execution, Deployment」→「Build Tools」→「Maven」
3. Maven home directoryが設定されているか確認
   （IntelliJ IDEAにバンドルされているMavenが使われます）
```

## 5. ビルドツール (Maven)

### 5.1 Mavenとは何か

**Maven（メイヴン）** は、Javaプロジェクトの「材料調達係」兼「組み立て係」です。

**Mavenの役割**:

1. **依存関係の管理**
   - 「Spring Boot使いたい」→ Mavenが自動でダウンロード
   - 必要なライブラリが100個あっても、Mavenが全部集めてくれる

2. **ビルドの自動化**
   - コンパイル、テスト、パッケージ作成を1コマンドで実行
   - `mvn package` と打つだけでアプリが完成

3. **プロジェクト構造の標準化**
   - どのMavenプロジェクトも同じフォルダ構成
   - 新しいプロジェクトでも迷わない

**重要なファイル: pom.xml**
- プロジェクトの「設計図」
- 必要なライブラリ（依存関係）を記載
- ビルド方法を定義

### 5.2 Mavenのインストール

#### 5.2.1 IntelliJ IDEAを使う場合（推奨）

**IntelliJ IDEAには既にMavenがバンドルされています**。別途インストール不要です。

確認方法:
```
IntelliJ IDEA → Settings → Build, Execution, Deployment → Build Tools → Maven
```

#### 5.2.2 スタンドアロン版のインストール（任意）

**Windows**:
1. [Maven ダウンロードページ](https://maven.apache.org/download.cgi) から `apache-maven-3.9.x-bin.zip` をダウンロード
2. `C:\Program Files\Apache\Maven` に展開
3. 環境変数設定:
   ```
   MAVEN_HOME = C:\Program Files\Apache\Maven\apache-maven-3.9.x
   Path に追加: %MAVEN_HOME%\bin
   ```

**macOS (Homebrew)**:
```bash
brew install maven
```

**Linux (Ubuntu/Debian)**:
```bash
sudo apt install maven -y
```

### 5.3 インストール確認

```bash
mvn -version

# 期待される出力例:
# Apache Maven 3.9.5 (57804ffe001d7215b5e7bcb531cf83df38f93546)
# Maven home: /opt/homebrew/Cellar/maven/3.9.5/libexec
# Java version: 17.0.9, vendor: Homebrew
```

## 6. データベース (PostgreSQL)

### 6.1 PostgreSQLとは何か

**PostgreSQL（ポストグレスキューエル）** は、データを保存・管理する「倉庫システム」です。

**何を保存するのか**:
- メモのタイトル
- メモの内容
- 作成日時、更新日時

**なぜPostgreSQLか**:
- ✅ 無料で使える（オープンソース）
- ✅ エンタープライズ（企業の業務システム）で広く使われている
- ✅ 機能が豊富（JSON型、全文検索など）
- ✅ SQL標準に準拠している

### 6.2 インストール方法の選択

PostgreSQLのインストールには2つの方法があります：

| 方法 | メリット | デメリット | 推奨度 |
|------|---------|-----------|--------|
| **Docker** | 簡単に起動・停止できる、複数バージョンを共存可能 | Dockerの理解が必要 | ⭐⭐⭐ 強く推奨 |
| **直接インストール** | Dockerが不要 | アンインストールが面倒、バージョン管理が難しい | ⭐⭐ 可 |

**推奨**: Docker を使う方法（次のセクションで説明）

### 6.3 PostgreSQL直接インストール（Dockerを使わない場合）

#### 6.3.1 Windows

1. [PostgreSQL ダウンロードページ](https://www.postgresql.org/download/windows/) にアクセス
2. "Download the installer" をクリック
3. PostgreSQL 15 の Windows x86-64 版をダウンロード
4. インストーラーを実行:
   ```
   - Installation Directory: デフォルトのまま
   - Components: すべて選択（PostgreSQL Server、pgAdmin 4など）
   - Data Directory: デフォルトのまま
   - Password: postgres ユーザーのパスワードを設定（覚えておく！）
   - Port: 5432（デフォルト）
   - Locale: Japanese, Japan
   ```

#### 6.3.2 macOS (Homebrew)

```bash
# PostgreSQLをインストール
brew install postgresql@15

# 自動起動設定
brew services start postgresql@15

# データベース作成
createdb memoapp
```

#### 6.3.3 Linux (Ubuntu/Debian)

```bash
# リポジトリ追加
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# インストール
sudo apt update
sudo apt install postgresql-15 -y

# サービス開始
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 6.4 PostgreSQL確認

```bash
# バージョン確認
psql --version

# 期待される出力例:
# psql (PostgreSQL) 15.5
```

## 7. Docker

### 7.1 Dockerとは何か

**Docker（ドッカー）** は、アプリケーションを「コンテナ」という箱に入れて動かす技術です。

**コンテナとは**:
- アプリケーションとその動作に必要なものをすべて詰め込んだ「パッケージ」
- どの環境（Windows、macOS、Linux）でも同じように動く

**例え話**:
- **従来**: 引っ越しで家具をバラバラに運ぶ（大変！）
- **Docker**: すべてをコンテナ（箱）に詰めて運ぶ（簡単！）

**PostgreSQLをDockerで動かすメリット**:
- ✅ インストールが簡単（1コマンドで起動）
- ✅ 不要になったら簡単に削除できる
- ✅ 複数バージョンを同時に動かせる
- ✅ 本番環境と同じ環境を作れる

### 7.2 Dockerのインストール

#### 7.2.1 Windows

1. **Docker Desktop for Windows** をダウンロード
   - [Docker Desktop ダウンロードページ](https://www.docker.com/products/docker-desktop/) にアクセス
   - "Download for Windows" をクリック

2. **システム要件確認**
   - Windows 10/11 64bit (Pro, Enterprise, Education)
   - WSL 2が有効になっていること

3. **インストール**
   - ダウンロードした `Docker Desktop Installer.exe` を実行
   - "Use WSL 2 instead of Hyper-V" を選択（推奨）
   - インストール完了後、再起動

4. **起動確認**
   - Docker Desktopを起動
   - タスクバーにDockerアイコンが表示されればOK

#### 7.2.2 macOS

1. **Docker Desktop for Mac** をダウンロード
   - [Docker Desktop ダウンロードページ](https://www.docker.com/products/docker-desktop/) にアクセス
   - Apple Silicon (M1/M2) または Intel を選択してダウンロード

2. **インストール**
   - ダウンロードした `.dmg` ファイルを開く
   - Docker.appをApplicationsフォルダにドラッグ
   - Applicationsフォルダから Docker を起動

3. **初回設定**
   - Docker Desktopが起動したらセットアップウィザードに従う

#### 7.2.3 Linux (Ubuntu)

```bash
# 古いバージョンを削除（存在する場合）
sudo apt-get remove docker docker-engine docker.io containerd runc

# 必要なパッケージをインストール
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release

# Dockerの公式GPGキーを追加
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# リポジトリ追加
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Docker Engineをインストール
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 現在のユーザーをdockerグループに追加（sudo不要にする）
sudo usermod -aG docker $USER

# ログアウト・ログインして設定を反映
```

### 7.3 Dockerインストール確認

```bash
# Dockerバージョン確認
docker --version

# 期待される出力例:
# Docker version 24.0.7, build afdd53b

# Docker Composeバージョン確認
docker compose version

# 期待される出力例:
# Docker Compose version v2.23.3

# 動作確認（Hello Worldコンテナを実行）
docker run hello-world

# 期待される出力:
# Hello from Docker!
# This message shows that your installation appears to be working correctly.
```

### 7.4 PostgreSQLコンテナの起動

このプロジェクトでは、PostgreSQLをDockerコンテナとして起動します。

**後ほど `docker-compose.yml` ファイルを作成しますが、今はコマンドで試してみましょう**:

```bash
# PostgreSQLコンテナを起動
docker run --name memoapp-postgres \
  -e POSTGRES_USER=memoapp_user \
  -e POSTGRES_PASSWORD=memoapp_pass \
  -e POSTGRES_DB=memoapp \
  -p 5432:5432 \
  -d postgres:15

# 起動確認
docker ps

# 期待される出力:
# CONTAINER ID   IMAGE         COMMAND                  CREATED         STATUS         PORTS                    NAMES
# xxxxxxxxxxxx   postgres:15   "docker-entrypoint.s…"   5 seconds ago   Up 4 seconds   0.0.0.0:5432->5432/tcp   memoapp-postgres
```

**コマンドの説明**:
- `--name memoapp-postgres`: コンテナに名前を付ける
- `-e POSTGRES_USER=...`: 環境変数を設定（データベースユーザー名）
- `-e POSTGRES_PASSWORD=...`: データベースパスワード
- `-e POSTGRES_DB=memoapp`: 最初に作成するデータベース名
- `-p 5432:5432`: ポート番号をマッピング（ホスト:コンテナ）
- `-d`: バックグラウンドで実行
- `postgres:15`: 使用するDockerイメージ（PostgreSQL 15）

**停止・削除方法**:
```bash
# コンテナ停止
docker stop memoapp-postgres

# コンテナ削除
docker rm memoapp-postgres

# 再起動
docker start memoapp-postgres
```

## 8. Git

### 8.1 Gitとは何か

**Git（ギット）** は、プログラムの変更履歴を記録する「タイムマシン」です。

**Gitでできること**:
- 📝 ファイルの変更履歴を記録
- 🔄 過去のバージョンに戻す
- 👥 複数人で同時に開発
- 🌿 新機能を別ブランチで開発

**GitHubとの違い**:
- **Git**: ローカル（自分のPC）で動くツール
- **GitHub**: Gitのリポジトリをクラウドで管理するサービス

### 8.2 Gitのインストール

#### 8.2.1 Windows

1. [Git for Windows](https://git-scm.com/download/win) からインストーラーをダウンロード
2. インストーラーを実行
3. すべてデフォルト設定でOK（特に変更不要）

#### 8.2.2 macOS

**方法1: Homebrew（推奨）**
```bash
brew install git
```

**方法2: Xcodeコマンドラインツール**
```bash
xcode-select --install
```

#### 8.2.3 Linux (Ubuntu/Debian)

```bash
sudo apt install git -y
```

### 8.3 Git初期設定

```bash
# ユーザー名を設定（GitHubと同じ名前推奨）
```

## 12. トラブルシューティング

### 12.1 Java関連

#### 問題: `java: command not found`

**原因**: 環境変数PATHが正しく設定されていない

**解決方法**:
```bash
# JAVA_HOMEが設定されているか確認
echo $JAVA_HOME  # macOS/Linux

# 設定されていない場合
export JAVA_HOME=/path/to/jdk-17
export PATH=$JAVA_HOME/bin:$PATH
```

#### 問題: `Unsupported class file major version XX`

**原因**: プロジェクトがJava 17を要求しているが、古いバージョンのJavaを使っている

**解決方法**: JDK 17をインストールして、JAVA_HOMEを更新

### 12.2 Maven関連

#### 問題: `mvn: command not found`

**原因**: Mavenがインストールされていない、またはPATHが通っていない

**解決方法**: IntelliJ IDEA内蔵のMavenを使う（IntelliJ IDEAのターミナルから実行）

#### 問題: 依存関係のダウンロードに失敗

**原因**: ネットワーク接続の問題、またはMavenリポジトリの問題

**解決方法**:
```bash
# キャッシュをクリアして再実行
mvn clean install -U
```

### 12.3 Docker関連

#### 問題: `Cannot connect to the Docker daemon`

**原因**: Dockerが起動していない

**解決方法**:
- Windows/macOS: Docker Desktopを起動
- Linux: `sudo systemctl start docker`

#### 問題: ポート5432が既に使用されている

**原因**: 別のPostgreSQLインスタンスが既に動いている

**解決方法**:
```bash
# 既存のPostgreSQLを停止
sudo systemctl stop postgresql  # Linux
brew services stop postgresql   # macOS

# または、別のポートを使う（docker-compose.ymlを編集）
ports:
  - "5433:5432"  # ホスト側を5433に変更
```

### 12.4 PostgreSQL関連

#### 問題: `FATAL: password authentication failed`

**原因**: パスワードが間違っている

**解決方法**: `application.yml` のパスワードとDocker起動時のパスワードを確認

#### 問題: `connection refused`

**原因**: PostgreSQLコンテナが起動していない

**解決方法**:
```bash
# コンテナが起動しているか確認
docker ps | grep postgres

# 起動していない場合
docker compose up -d
```

### 12.5 アプリケーション起動関連

#### 問題: `Port 8080 is already in use`

**原因**: 別のアプリケーションがポート8080を使用している

**解決方法**:
```bash
# 使用中のプロセスを確認
# Windows
netstat -ano | findstr :8080

# macOS/Linux
lsof -i :8080

# プロセスを停止するか、別のポートを使う（application.ymlを編集）
server:
  port: 8081
```

#### 問題: `Flyway migration failed`

**原因**: マイグレーションスクリプトにエラーがある、または既にテーブルが存在する

**解決方法**:
```bash
# Flywayの状態確認
mvn flyway:info

# 問題がある場合は修復
mvn flyway:repair

# データベースを完全にリセット（注意：データが消えます）
mvn flyway:clean
mvn flyway:migrate
```
