# 学習用Webアプリケーション開発ロードマップ

Java/Kotlin + Spring Boot + JavaScript/React/Next.js を軸に、「従来構成」と「モダン構成」の比較学習を通じて、実践的なスキル習得を目指すための学習ロードマップです。

## 🎯 このロードマップの目的

- **段階的な学習**: シンプルなCRUDから複雑な業務システムまで、3段階で学習
- **技術の比較**: 従来型とモダン型を実装し、それぞれのメリット・デメリットを体感
- **実務への応用**: 実際の業務システム開発で必要となる知識とスキルを習得

---

## ステップ1：メモ帳アプリ（基本のCRUD）

まずは、Webアプリケーションの基本であるCRUD（作成・読み取り・更新・削除）を網羅したシンプルなアプリケーションを作成し、両構成の根本的な違いを体感します。

### 📋 テーマ
- 基本的なCRUD操作
- SPAとMVCの比較
- REST APIの基礎

### 🎓 学習目的
- サーバーサイドレンダリング (MVC) とクライアントサイドレンダリング (SPA) の違いを理解する
- モノリシックな構成と、API分離型（Headless）構成の違いを理解する
- 両構成におけるデータ処理の流れを掴む
- TypeScriptによる型安全な開発を体験する

### 💻 作成するバージョン

#### v1 (従来型) - リポジトリ: [learn-memo-app-traditional](https://github.com/fukt-dev/learn-memo-app-traditional)
- **技術スタック**: Java + Spring Boot + Thymeleaf + JavaScript (Vanilla)
- **アーキテクチャ**: サーバーサイドレンダリング（MVC）
- **特徴**:
  - サーバー側で画面（HTML）を生成
  - 画面遷移時にページ全体をリロード
  - MyBatisによるSQL直接記述
- **学習ポイント**:
  - MVCパターンの基礎
  - Thymeleafテンプレートエンジン
  - フォームベースの画面遷移
  - MyBatisの2-way SQL

#### v2 (モダン型) - リポジトリ: [learn-memo-app-modern] (https://github.com/fukt-dev/learn-memo-app-modern (プライベートリポジトリ))
- **技術スタック**: Kotlin + Spring Boot + React + TypeScript
- **アーキテクチャ**: SPA（Single Page Application）+ REST API
- **特徴**:
  - バックエンドはJSON APIのみを提供
  - フロントエンドで画面を動的に生成
  - 画面遷移時にリロード不要
  - Spring Data JPAによるO/Rマッピング
- **学習ポイント**:
  - REST API設計
  - Reactコンポーネント設計
  - TypeScriptによる型安全性
  - Axiosを使ったHTTP通信
  - Spring Data JPAの基本

### 📊 難易度
⭐ (初級)

### 🔗 関連リポジトリ
- 従来型: https://github.com/fukt-dev/learn-memo-app-traditional
- モダン型: https://github.com/fukt-dev/learn-memo-app-modern (プライベートリポジトリ)

## ステップ2：タスク管理アプリ（リレーションの導入）

メモ帳アプリ（単一エンティティ）から一歩進め、「1対多」のデータ関係（リレーション）を扱います。これにより、より実務に近いデータ構造の扱いに慣れます。

### 📋 テーマ
- 「1対多」のリレーション
- フロントエンドの状態管理
- ReactとNext.jsの違い

### 🎓 学習目的
- 「プロジェクト」と「タスク」のような、親（1）と子（多）の関係を持つデータの扱い方を学ぶ
- フロントエンド側で関連する複数のデータをどのように保持し、表示に反映させるか（状態管理）を学ぶ
- 複雑なUIにおける従来型とモダン型の開発コストの違いを体感する
- ReactからNext.jsへのステップアップを体験する

### 💡 ステップ1からの難易度上昇ポイント
- データモデルが複数になる（ProjectテーブルとTaskテーブル）
- リレーションを考慮したAPI設計が必要
- UIが複雑になる（例：「プロジェクトAに紐づくタスク一覧」の表示）
- 状態管理の重要性が増す（親子関係のデータ同期）

### 💻 作成するバージョン

#### v1 (従来型) - リポジトリ: 未作成（予定）
- **技術スタック**: Java + Spring Boot + Thymeleaf + JavaScript (Vanilla)
- **アーキテクチャ**: サーバーサイドレンダリング（MVC）
- **特徴**:
  - サーバー側で親子関係を含むデータを取得してHTMLを生成
  - フォーム送信による画面遷移
  - JavaScriptによる動的なUI制御（タスクの追加・削除など）
- **学習ポイント**:
  - MyBatisでのJOIN処理
  - Thymeleafでのネストしたデータの表示
  - 素のJavaScript（Vanilla JS）によるDOM操作
  - 従来型での動的UIの限界を体感

#### v2 (モダン型 - React) - リポジトリ: 未作成（予定）
- **技術スタック**: Kotlin + Spring Boot + React + TypeScript
- **アーキテクチャ**: SPA + REST API
- **特徴**:
  - 親子関係を持つデータをJSON APIで取得
  - Reactの状態管理（useState、useEffect）
  - コンポーネントの階層構造でデータを受け渡し
- **学習ポイント**:
  - Reactの状態管理（親子関係のデータ）
  - useContextやカスタムフックの活用
  - コンポーネント間のデータ受け渡し
  - リレーショナルデータのSPA表示

#### v3 (モダン型 - Next.js) - リポジトリ: 未作成（予定）
- **技術スタック**: Kotlin + Spring Boot + Next.js (App Router) + TypeScript
- **アーキテクチャ**: SSR/SSG + REST API
- **特徴**:
  - Next.jsのApp Routerを使用
  - Server ComponentsとClient Componentsの使い分け
  - ファイルベースルーティング
  - SEO対策（SSR/SSG）
- **学習ポイント**:
  - Next.jsの基本概念（App Router）
  - Server ComponentsとClient Componentsの違い
  - ReactとNext.jsの違いを実感
  - Next.jsのルーティングとレイアウト
  - SSRによるパフォーマンス最適化の基礎

### 📊 難易度
⭐⭐ (中級)

### 🎯 なぜ3バージョン作るのか？

このステップで3バージョンを作成する理由：

1. **React → Next.jsの段階的学習**
   - v2でReactの基礎をしっかり固める
   - v3でNext.jsの追加機能を学ぶ
   - 同じアプリを実装することで違いが明確になる

2. **フレームワークの価値を実感**
   - Next.jsのルーティング、レイアウト機能の便利さ
   - SSRによるSEO対策とパフォーマンス向上
   - ファイル構成の整理しやすさ

3. **実務での選択基準を理解**
   - どんな時にReactを選ぶべきか
   - どんな時にNext.jsを選ぶべきか
   - それぞれのメリット・デメリットを体感

## ステップ3：伝票処理アプリ（最終目標）

エンタープライズ開発（業務システム）の縮図とも言える、複雑なデータモデルとビジネスロジックを扱うアプリケーションに挑戦します。

### 📋 テーマ
- 複雑なデータモデル（多対多、マスタ・トランザクション）
- 動的で複雑なUI（動的フォーム）
- ビジネスロジックの実装と責務分担
- 実務レベルの業務システム開発

### 🎓 学習目的
- 「顧客マスタ」「商品マスタ」「伝票ヘッダ」「伝票明細」といった、複数のマスタとトランザクションデータが関連する複雑なデータモデルを設計・実装する
- 伝票明細行の動的な追加・削除や、合計金額のリアルタイム自動計算など、複雑なUIを実装する
- 「見積伝票を確定したら受注伝票を作成する」といった、単なるCRUDを超えたビジネスロジックを実装する
- ビジネスロジックをどの層（フロントエンド/バックエンド）に持たせるべきかを考察する
- トランザクション処理とエラーハンドリングを実装する

### 💡 ステップ2からの難易度上昇ポイント
- データモデルの複雑化（マスタ×複数、トランザクション×複数、多対多）
- 動的フォーム（明細行の追加・削除、自動計算）
- ビジネスルールの実装（在庫チェック、金額計算、ステータス管理）
- トランザクション管理（複数テーブルの整合性）
- 実務レベルのエラーハンドリング

### 💻 作成するバージョン

#### v1 (従来型) - リポジトリ: 未作成（予定）
- **技術スタック**: Java + Spring Boot + Thymeleaf + JavaScript (Vanilla)
- **アーキテクチャ**: サーバーサイドレンダリング（MVC）
- **特徴**:
  - サーバー側で複雑なデータを結合してHTMLを生成
  - フォーム送信による画面遷移
  - JavaScriptによる明細行の動的追加・削除
  - サーバー側で金額計算やバリデーション
- **学習ポイント**:
  - 従来型での複雑なフォーム処理の困難さを体感
  - 手書きDOM操作の限界（モダン構成への移行動機）
  - サーバーサイドでのトランザクション管理
  - MyBatisでの複雑なJOIN処理
  - セッション管理の重要性

#### v2 (モダン型 - Next.js) - リポジトリ: 未作成（予定）
- **技術スタック**: Kotlin + Spring Boot + Next.js (App Router) + TypeScript
- **アーキテクチャ**: SSR/SSG + REST API
- **特徴**:
  - バックエンドはビジネスロジックとデータ管理に専念
  - フロントエンドで複雑なフォームUIを実装
  - React Hook FormやZodを使ったバリデーション
  - リアルタイムな金額計算と表示更新
  - Next.jsの機能を活用した最適化
- **学習ポイント**:
  - Next.jsでの複雑なフォーム実装
  - React Hook Formによるフォーム管理
  - Zodによるバリデーション
  - フロントエンドとバックエンドの責務分担
  - 楽観的ロック（Optimistic Locking）
  - Next.jsのServer Actionsの活用（検討）
  - エラーハンドリングとユーザーフィードバック

### 📊 難易度
⭐⭐⭐ (上級)

### 🎯 なぜ2バージョンに絞るのか？

ステップ3で2バージョンに絞る理由：

1. **React版は不要**
   - ステップ2でReactは十分に学習済み
   - 複雑なアプリではNext.jsの方が実務的
   - 同じ技術スタックの繰り返しは非効率

2. **従来型との違いを最大化**
   - 最も複雑なアプリで従来型とモダン型の差を体感
   - 動的フォームの実装難易度を比較
   - 保守性・拡張性の違いを実感

3. **実務レベルの技術習得**
   - Next.jsは現在最も需要が高いフレームワーク
   - Server ComponentsやServer Actionsなど最新技術を学ぶ
   - 実際の業務システムに近い構成

4. **学習効率の最大化**
   - 開発工数を削減し、品質向上に時間を投資
   - テスト、ドキュメント、リファクタリングに注力
   - より実務に近い開発体験

### 技術スタック選定の理由

#### なぜステップ2で3バージョン？
- **段階的学習**: React → Next.jsと段階的に学ぶことで、理解が深まる
- **比較学習**: 同じアプリを異なる技術で実装し、違いを明確に体感
- **適切な複雑さ**: タスク管理アプリは3バージョン作成しても負担が少ない

#### なぜステップ3で2バージョン？
- **効率重視**: Reactは既に習得済み、Next.jsに集中
- **実務志向**: 複雑な業務システムではNext.jsが実務標準
- **品質重視**: バージョン数を絞り、より質の高い実装に注力

### 学習の流れ

```
JavaScript → TypeScript → React → Next.js
(基礎)      (型安全)    (SPA)    (フレームワーク)

Java → Kotlin
(従来型) (モダン型)
```

### 推奨学習順序

1. **ステップ1の両バージョンを完成させる**
   - 従来型とモダン型の違いを理解する
   - TypeScriptの基礎を固める

2. **ステップ2のv1, v2, v3を順に実装**
   - まずv1で従来型のリレーション処理を学ぶ
   - v2でReactの状態管理をマスター
   - v3でNext.jsの機能を習得

3. **ステップ3のv1, v2を実装**
   - v1で従来型の限界を体感
   - v2で実務レベルのモダン開発を習得

   - Spring Boot: https://spring.io/projects/spring-boot
   - React: https://react.dev/
   - Next.js: https://nextjs.org/docs
   - TypeScript: https://www.typescriptlang.org/docs/

2. **コードを読む**
   - 既存のコードをしっかり読んで理解する
   - なぜそう書かれているのかを考える

   - わからないことは小さく切り出して試す
   - エラーメッセージをしっかり読む
