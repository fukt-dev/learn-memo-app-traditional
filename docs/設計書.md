# メモ帳アプリケーション 設計書（従来型）

## 1. システム概要

### 1.1 目的
本アプリケーションは、開発未経験者がWebアプリケーション開発の基礎を学ぶための教材として、基本的なCRUD操作を実装したメモ帳アプリケーションです。従来型のサーバーサイドレンダリング（MVC）構成を採用し、3層レイヤーアーキテクチャで実装します。

### 1.2 対象ユーザー
- Webアプリケーション開発の学習を始めたばかりの初心者
- 従来型のMVCアーキテクチャを学びたい開発者
- Java/Spring Bootの基礎を学びたい開発者

### 1.3 主な機能
- メモの新規作成
- メモの一覧表示
- メモの詳細表示
- メモの編集
- メモの削除
- メモの検索（オプション）

## 2. アーキテクチャ設計

### 2.1 3層レイヤーアーキテクチャ

本アプリケーションは、以下の3層構造で実装します：

```
┌─────────────────────────────────────┐
│    プレゼンテーション層              │
│  (Presentation Layer)               │
│  - Controller                       │
│  - JSP/Thymeleaf (View)             │
│  - JavaScript/jQuery (動的な操作)   │
└─────────────────────────────────────┘
              ↓↑
┌─────────────────────────────────────┐
│    ビジネスロジック層                │
│  (Business Logic Layer)             │
│  - Service                          │
│  - DTO (Data Transfer Object)       │
└─────────────────────────────────────┘
              ↓↑
┌─────────────────────────────────────┐
│    データアクセス層                  │
│  (Data Access Layer)                │
│  - Repository/Mapper                │
│  - Entity                           │
│  - Database (MySQL/PostgreSQL)      │
└─────────────────────────────────────┘
```

#### 2.1.1 プレゼンテーション層
- **責務**: ユーザーとのインタラクション、画面表示、入力値の受け取り
- **主要コンポーネント**:
  - **Controller**: HTTPリクエストを受け取り、Serviceを呼び出し、Viewに渡すデータを準備
  - **JSP/Thymeleaf**: HTMLを生成するテンプレートエンジン
  - **JavaScript/jQuery**: クライアントサイドでの動的な操作（確認ダイアログ、バリデーション表示など）

#### 2.1.2 ビジネスロジック層
- **責務**: ビジネスルールの実装、トランザクション管理、データの加工
- **主要コンポーネント**:
  - **Service**: ビジネスロジックを実装するクラス
  - **DTO**: 層間でデータを受け渡すためのオブジェクト

#### 2.1.3 データアクセス層
- **責務**: データベースへのアクセス、永続化処理
- **主要コンポーネント**:
  - **Repository/Mapper**: データベース操作を抽象化するインターフェース
  - **Entity**: データベーステーブルに対応するJavaオブジェクト
  - **Database**: 実際のデータ格納先（MySQL or PostgreSQL）

### 2.2 アーキテクチャ選択の理由

#### なぜ3層アーキテクチャか？
1. **関心の分離**: 各層が明確な責務を持つことで、コードの理解と保守が容易
2. **テスタビリティ**: 各層を独立してテストできる
3. **再利用性**: ビジネスロジックを複数の画面から利用可能
4. **学習のしやすさ**: 役割が明確で初心者にも理解しやすい

#### なぜサーバーサイドレンダリング（MVC）か？
1. **シンプルさ**: クライアント側の複雑な状態管理が不要
2. **学習曲線**: フロントエンドフレームワークの習得が不要
3. **従来型の理解**: モダン構成との比較学習のベースライン

## 3. 機能要件

### 3.1 機能一覧

| 機能ID | 機能名 | 優先度 | 説明 |
|--------|--------|--------|------|
| F001 | メモ一覧表示 | 必須 | 登録されているすべてのメモを一覧表示する |
| F002 | メモ詳細表示 | 必須 | 選択したメモの詳細を表示する |
| F003 | メモ新規作成 | 必須 | 新しいメモを作成する |
| F004 | メモ編集 | 必須 | 既存のメモを編集する |
| F005 | メモ削除 | 必須 | 既存のメモを削除する（削除確認あり） |
| F006 | メモ検索 | 推奨 | タイトルまたは内容でメモを検索する |
| F007 | ページング | オプション | 一覧表示時にページング機能を提供する |
| F008 | カテゴリー分類 | オプション | メモをカテゴリーで分類する |

### 3.2 機能詳細仕様

#### F001: メモ一覧表示
- **入力**: なし
- **処理**:
  1. データベースからすべてのメモを取得
  2. 作成日時の降順（新しい順）でソート
  3. 一覧画面に表示
- **出力**: メモ一覧（ID、タイトル、作成日時、更新日時）
- **例外処理**: データベース接続エラー時はエラーメッセージを表示

#### F002: メモ詳細表示
- **入力**: メモID
- **処理**:
  1. 指定されたIDのメモをデータベースから取得
  2. 詳細画面に表示
- **出力**: メモ詳細（ID、タイトル、内容、作成日時、更新日時）
- **例外処理**:
  - メモが存在しない場合: 404エラー
  - 不正なID形式: バリデーションエラー

#### F003: メモ新規作成
- **入力**: タイトル、内容
- **処理**:
  1. 入力値のバリデーション
  2. 現在日時を作成日時・更新日時として設定
  3. データベースに保存
  4. 一覧画面にリダイレクト
- **出力**: 作成成功メッセージ、一覧画面への遷移
- **バリデーション**:
  - タイトル: 必須、最大100文字
  - 内容: 任意、最大5000文字
- **例外処理**: バリデーションエラー時は入力値を保持して再表示

#### F004: メモ編集
- **入力**: メモID、タイトル、内容
- **処理**:
  1. 指定されたIDのメモをデータベースから取得
  2. 入力値のバリデーション
  3. 更新日時を現在日時に更新
  4. データベースを更新
  5. 詳細画面にリダイレクト
- **出力**: 更新成功メッセージ、詳細画面への遷移
- **バリデーション**: F003と同じ
- **例外処理**:
  - メモが存在しない場合: 404エラー
  - バリデーションエラー時は入力値を保持して再表示

#### F005: メモ削除
- **入力**: メモID
- **処理**:
  1. JavaScriptで削除確認ダイアログを表示
  2. ユーザーがOKを押した場合のみ削除リクエストを送信
  3. 指定されたIDのメモをデータベースから削除
  4. 一覧画面にリダイレクト
- **出力**: 削除成功メッセージ、一覧画面への遷移
- **例外処理**: メモが存在しない場合でもエラーとせず、一覧画面にリダイレクト

#### F006: メモ検索（推奨機能）
- **入力**: 検索キーワード
- **処理**:
  1. タイトルまたは内容に検索キーワードを含むメモを取得
  2. 一覧画面に検索結果を表示
- **出力**: 検索結果のメモ一覧
- **例外処理**: 検索結果が0件の場合は「該当するメモがありません」を表示

## 4. 非機能要件

### 4.1 パフォーマンス要件
- 一覧画面の初期表示: 3秒以内
- CRUD操作のレスポンス: 1秒以内
- 同時アクセス: 10ユーザーまで（学習用のため）

### 4.2 可用性要件
- 学習用のため、高可用性は求めない
- ローカル環境での動作を前提

### 4.3 セキュリティ要件
- XSS対策: テンプレートエンジンのエスケープ機能を使用
- SQLインジェクション対策: プリペアドステートメント使用
- CSRF対策: Spring Securityのデフォルト設定を使用（将来的に）
- 認証・認可: 現バージョンでは実装しない（将来の拡張として）

### 4.4 保守性要件
- コードの可読性: 初心者が理解できるシンプルなコード
- コメント: 重要な処理には日本語コメントを記載
- ログ出力: エラー発生時の原因特定が容易なログ設計

### 4.5 移植性要件
- OS: Windows、macOS、Linuxで動作
- データベース: MySQL 8.0以上、またはPostgreSQL 12以上
- Java: Java 17以上
- ブラウザ: Chrome、Firefox、Edge、Safari（最新版）

## 5. データベース設計

### 5.1 ER図

```
┌─────────────────────┐
│      memos          │
├─────────────────────┤
│ id (PK)             │ BIGINT
│ title               │ VARCHAR(100)
│ content             │ TEXT
│ created_at          │ TIMESTAMP
│ updated_at          │ TIMESTAMP
└─────────────────────┘
```

### 5.2 テーブル定義

#### memosテーブル

| カラム名 | データ型 | NULL許可 | デフォルト値 | 説明 |
|----------|----------|----------|--------------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主キー、メモID |
| title | VARCHAR(100) | NO | - | メモのタイトル |
| content | TEXT | YES | NULL | メモの内容 |
| created_at | TIMESTAMP | NO | CURRENT_TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新日時 |

### 5.3 インデックス設計

| インデックス名 | カラム | 種類 | 目的 |
|---------------|--------|------|------|
| PRIMARY | id | PRIMARY KEY | 主キー |
| idx_created_at | created_at | INDEX | 一覧表示時のソート用 |
| idx_title_content | title, content | FULLTEXT | 検索機能用（オプション） |

### 5.4 DDL（テーブル作成SQL）

```sql
-- memosテーブル作成
CREATE TABLE memos (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    content TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_created_at (created_at DESC),
    FULLTEXT INDEX idx_title_content (title, content) -- MySQL 5.6以上、オプション機能用
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 7.2 パッケージ構成の説明

#### controller（プレゼンテーション層）
- HTTPリクエストを受け取り、Serviceを呼び出す
- ビジネスロジックは持たない
- Viewに渡すデータの準備のみを行う

#### service（ビジネスロジック層）
- アプリケーションのビジネスルールを実装
- トランザクション境界を管理（@Transactional）
- Repositoryを呼び出してデータを永続化

#### repository（データアクセス層）
- データベースへのアクセスを抽象化
- Spring Data JPAを使用してCRUD操作を簡潔に実装

#### entity（エンティティ）
- データベーステーブルに対応するJavaクラス
- JPA アノテーションでマッピング定義

#### dto（データ転送オブジェクト）
- 層間でデータを受け渡すためのクラス
- Entityとは別に定義することで、層間の独立性を保つ

```java
package com.example.memoapp.entity;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import java.time.LocalDateTime;

/**
 * メモエンティティ
 * memosテーブルに対応するJPAエンティティ
 */
@Entity
@Table(name = "memos")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Memo {

    /** メモID（主キー） */
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    /** タイトル（必須、最大100文字） */
    @Column(nullable = false, length = 100)
    private String title;

    /** 内容（任意、TEXT型） */
    @Column(columnDefinition = "TEXT")
    private String content;

    /** 作成日時 */
    @Column(name = "created_at", nullable = false, updatable = false)
    private LocalDateTime createdAt;

    /** 更新日時 */
    @Column(name = "updated_at", nullable = false)
    private LocalDateTime updatedAt;

    /**
     * エンティティ保存前に実行される処理
     * 作成日時と更新日時を自動設定
     */
    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    /**
     * エンティティ更新前に実行される処理
     * 更新日時を自動設定
     */
    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }
}
```

#### MemoServiceImpl.java
```java
package com.example.memoapp.service.impl;

import com.example.memoapp.dto.MemoDto;
import com.example.memoapp.entity.Memo;
import com.example.memoapp.exception.MemoNotFoundException;
import com.example.memoapp.repository.MemoRepository;
import com.example.memoapp.service.MemoService;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;
import java.util.stream.Collectors;

/**
 * メモサービス実装クラス
 */
@Service
@RequiredArgsConstructor
@Transactional(readOnly = true)
public class MemoServiceImpl implements MemoService {

    private final MemoRepository memoRepository;

    @Override
    public List<MemoDto> findAll() {
        return memoRepository.findAllByOrderByCreatedAtDesc()
                .stream()
                .map(this::convertToDto)
                .collect(Collectors.toList());
    }

    @Override
    public MemoDto findById(Long id) {
        Memo memo = memoRepository.findById(id)
                .orElseThrow(() -> new MemoNotFoundException("メモが見つかりません。ID: " + id));
        return convertToDto(memo);
    }

    @Override
    public List<MemoDto> searchByKeyword(String keyword) {
        if (keyword == null || keyword.isEmpty()) {
            return findAll();
        }
        return memoRepository.searchByKeyword(keyword)
                .stream()
                .map(this::convertToDto)
                .collect(Collectors.toList());
    }

    @Override
    @Transactional
    public MemoDto create(MemoDto memoDto) {
        Memo memo = convertToEntity(memoDto);
        Memo savedMemo = memoRepository.save(memo);
        return convertToDto(savedMemo);
    }

    @Override
    @Transactional
    public MemoDto update(Long id, MemoDto memoDto) {
        Memo existingMemo = memoRepository.findById(id)
                .orElseThrow(() -> new MemoNotFoundException("メモが見つかりません。ID: " + id));

        existingMemo.setTitle(memoDto.getTitle());
        existingMemo.setContent(memoDto.getContent());

        Memo updatedMemo = memoRepository.save(existingMemo);
        return convertToDto(updatedMemo);
    }

    @Override
    @Transactional
    public void delete(Long id) {
        memoRepository.deleteById(id);
    }

    /**
     * EntityをDTOに変換
     */
    private MemoDto convertToDto(Memo memo) {
        return new MemoDto(
                memo.getId(),
                memo.getTitle(),
                memo.getContent(),
                memo.getCreatedAt(),
                memo.getUpdatedAt()
        );
    }

    /**
     * DTOをEntityに変換
     */
    private Memo convertToEntity(MemoDto dto) {
        Memo memo = new Memo();
        memo.setTitle(dto.getTitle());
        memo.setContent(dto.getContent());
        return memo;
    }
}
```

```java
package com.example.memoapp.controller;

import com.example.memoapp.dto.MemoDto;
import com.example.memoapp.service.MemoService;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.validation.BindingResult;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.servlet.mvc.support.RedirectAttributes;

/**
 * メモコントローラー
 * メモに関する画面リクエストを処理
 */
@Controller
@RequestMapping("/memos")
@RequiredArgsConstructor
public class MemoController {

    private final MemoService memoService;

    /**
     * メモ一覧画面表示
     */
    @GetMapping
    public String list(@RequestParam(required = false) String keyword, Model model) {
        if (keyword != null && !keyword.isEmpty()) {
            model.addAttribute("memos", memoService.searchByKeyword(keyword));
            model.addAttribute("keyword", keyword);
        } else {
            model.addAttribute("memos", memoService.findAll());
        }
        return "memos/list";
    }

    /**
     * メモ詳細画面表示
     */
    @GetMapping("/{id}")
    public String detail(@PathVariable Long id, Model model) {
        model.addAttribute("memo", memoService.findById(id));
        return "memos/detail";
    }

    /**
     * メモ新規作成フォーム表示
     */
    @GetMapping("/new")
    public String newForm(Model model) {
        model.addAttribute("memoDto", new MemoDto());
        return "memos/new";
    }

    /**
     * メモ新規作成処理
     */
    @PostMapping
    public String create(@Valid @ModelAttribute MemoDto memoDto,
                        BindingResult bindingResult,
                        RedirectAttributes redirectAttributes) {
        if (bindingResult.hasErrors()) {
            return "memos/new";
        }

        memoService.create(memoDto);
        redirectAttributes.addFlashAttribute("message", "メモを作成しました");
        return "redirect:/memos";
    }

    /**
     * メモ編集フォーム表示
     */
    @GetMapping("/{id}/edit")
    public String editForm(@PathVariable Long id, Model model) {
        model.addAttribute("memoDto", memoService.findById(id));
        return "memos/edit";
    }

    /**
     * メモ更新処理
     */
    @PostMapping("/{id}/update")
    public String update(@PathVariable Long id,
                        @Valid @ModelAttribute MemoDto memoDto,
                        BindingResult bindingResult,
                        RedirectAttributes redirectAttributes) {
        if (bindingResult.hasErrors()) {
            memoDto.setId(id);
            return "memos/edit";
        }

        memoService.update(id, memoDto);
        redirectAttributes.addFlashAttribute("message", "メモを更新しました");
        return "redirect:/memos/" + id;
    }

    /**
     * メモ削除処理
     */
    @PostMapping("/{id}/delete")
    public String delete(@PathVariable Long id, RedirectAttributes redirectAttributes) {
        memoService.delete(id);
        redirectAttributes.addFlashAttribute("message", "メモを削除しました");
        return "redirect:/memos";
    }
}
```

本設計書は、開発未経験者が従来型のWebアプリケーション開発を学ぶための教材として、メモ帳アプリケーションの設計を詳細に記述しました。

### 本設計書のポイント

1. **3層レイヤーアーキテクチャ**
   - 関心の分離により、コードの理解と保守が容易
   - 各層の責務が明確

2. **Spring Bootのベストプラクティス**
   - 設定より規約（Convention over Configuration）
   - 依存性注入（DI）によるテスタビリティの向上

3. **段階的な学習**
   - まずは基本的なCRUD機能から
   - オプション機能で徐々にステップアップ

4. **モダン構成への橋渡し**
   - 従来型を理解することで、モダン構成（v2: React + REST API）との比較が可能
   - なぜモダン構成が必要なのかを実感できる

### 次のステップ

本設計書に基づいてメモ帳アプリを実装したら、次は以下に挑戦してください：

## 付録

### A. 用語集

- **CRUD**: Create（作成）、Read（読み取り）、Update（更新）、Delete（削除）の頭文字
- **MVC**: Model-View-Controller の略。MVCパターンはアプリケーションを3つの役割に分離する設計パターン
- **DTO**: Data Transfer Object。層間でデータを受け渡すためのオブジェクト
- **Entity**: データベーステーブルに対応するJavaオブジェクト
- **Repository**: データアクセスを抽象化するパターン
- **DI**: Dependency Injection（依存性注入）。オブジェクトの依存関係を外部から注入する設計パターン
- **ORM**: Object-Relational Mapping。オブジェクトとリレーショナルデータベースをマッピングする技術
