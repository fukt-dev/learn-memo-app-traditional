## 1. はじめに

### 1.1 このガイドについて

本ガイドは、メモ帳アプリケーションを開発する際の **実践的な手順** を説明します。

**前提知識**:
- 環境構築が完了していること（[環境構築手順.md](./環境構築手順.md) 参照）
- 設計書を一読していること（[設計書.md](./設計書.md) 参照）

### 1.2 学習の進め方

```
1. コードを読む → 2. 写経する → 3. 動かす → 4. 改造する
```

**推奨学習フロー**:
1. **まず動かす**: サンプルコードをそのまま動かして動作を確認
2. **コードを読む**: 各行が何をしているか理解する
3. **小さく変える**: メッセージや数値を変えてみる
4. **新機能を追加**: 理解が深まったら新しい機能に挑戦

## 2. プロジェクト構造の理解

### 2.1 全体像

```
learn-memo-app-traditional/
├── src/                          ← ソースコードがここにある
│   ├── main/                     ← アプリケーション本体
│   │   ├── java/                 ← Javaコード
│   │   │   └── com/example/memoapp/
│   │   │       ├── controller/   ← 【プレゼンテーション層】画面リクエストを処理
│   │   │       ├── service/      ← 【ビジネスロジック層】業務ルールを実装
│   │   │       ├── mapper/       ← 【データアクセス層】DBアクセス
│   │   │       ├── entity/       ← データベーステーブルに対応するクラス
│   │   │       ├── dto/          ← 画面とサービス間でデータをやり取りするクラス
│   │   │       └── exception/    ← エラー処理
│   │   └── resources/            ← 設定ファイルとテンプレート
│   │       ├── application.yml   ← アプリケーション設定
│   │       ├── templates/        ← Thymeleaf HTML テンプレート
│   │       ├── static/           ← CSS、JavaScript、画像
│   │       ├── mybatis/mapper/   ← MyBatis SQL定義（2waySQL）
│   │       └── db/migration/     ← Flyway マイグレーションSQL
│   └── test/                     ← テストコード
│       └── java/                 ← Javaテストコード
├── docs/                         ← ドキュメント
│   ├── 設計書.md
│   ├── 環境構築手順.md
│   └── 開発ガイド.md (本ファイル)
├── pom.xml                       ← Maven設定ファイル（依存関係定義）
├── docker-compose.yml            ← Docker設定（PostgreSQL起動）
└── README.md                     ← プロジェクト概要
```

### 2.2 各ディレクトリの役割

#### 2.2.1 `src/main/java` - Javaソースコード

| ディレクトリ | 役割 | 例 |
|-------------|------|---|
| `controller/` | **HTTPリクエストを受け取る** | ボタンをクリック → Controllerが反応 |
| `service/` | **ビジネスロジックを実装** | メモを保存する前にバリデーション |
| `mapper/` | **データベースにアクセス** | SELECT, INSERT, UPDATE, DELETE |
| `entity/` | **データベースの1行を表現** | memosテーブルの1行 = Memoオブジェクト |
| `dto/` | **画面とサービス間でデータ転送** | 入力フォームのデータを受け取る |
| `exception/` | **エラーを処理** | メモが見つからない → 404エラー |

#### 2.2.2 `src/main/resources` - リソースファイル

| ファイル/ディレクトリ | 役割 | 変更頻度 |
|---------------------|------|---------|
| `application.yml` | アプリケーション設定（DB接続情報など） | 少ない |
| `templates/` | Thymeleaf HTMLテンプレート | 多い |
| `static/css/` | スタイルシート（デザイン） | 中程度 |
| `static/js/` | JavaScript（動的な動作） | 中程度 |
| `mybatis/mapper/` | SQLファイル（2waySQL） | 中程度 |
| `db/migration/` | データベース変更スクリプト | 少ない（追加のみ） |

#### 2.2.3 `pom.xml` - Maven設定ファイル

**何をするファイルか**:
- 使いたいライブラリ（Spring Boot、MyBatisなど）を宣言
- Mavenが自動でダウンロードしてくれる

**基本構造**:
```xml
<project>
  <groupId>com.example</groupId>        <!-- プロジェクトのグループ名 -->
  <artifactId>memo-app</artifactId>     <!-- プロジェクト名 -->
  <version>1.0.0-SNAPSHOT</version>     <!-- バージョン -->

  <dependencies>                        <!-- 使用するライブラリ -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <!-- 他のライブラリ... -->
  </dependencies>
</project>
```

### 2.3 データの流れ

```
【ブラウザ】
    ↓ HTTPリクエスト（例: GET /memos）
【Controller】
    ↓ Serviceを呼び出す
【Service】
    ↓ Mapperを呼び出す
【Mapper】
    ↓ SQLを実行
【PostgreSQL】
    ↑ データを返す
【Mapper】
    ↑ Entityオブジェクトに変換
【Service】
    ↑ DTOに変換
【Controller】
    ↑ Viewにデータを渡す
【Thymeleaf】
    ↑ HTMLを生成
【ブラウザ】
    画面表示！
```

## 3. 開発の流れ

### 3.1 新機能開発の基本ステップ

新しい機能（例: メモにカテゴリーを追加）を開発する場合の流れ：

```
ステップ1: データベース設計
  → Flywayマイグレーションファイル作成

ステップ2: データアクセス層
  → Entity修正
  → Mapperインターフェース修正
  → Mapper XML（2waySQL）修正

ステップ3: ビジネスロジック層
  → DTO修正
  → Service修正

ステップ4: プレゼンテーション層
  → Controller修正
  → Thymeleafテンプレート修正

ステップ5: テスト
  → 単体テスト作成
  → 動作確認
```

### 3.2 具体例: メモの新規作成機能の流れ

#### 3.2.1 データベース設計

```sql
-- db/migration/V1__create_memos_table.sql
CREATE TABLE memos (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    content TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.2 Entityクラス

```java
// entity/Memo.java
@Data
public class Memo {
    private Long id;          // データベースのid列に対応
    private String title;     // データベースのtitle列に対応
    private String content;   // データベースのcontent列に対応
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

#### 3.2.3 Mapperインターフェース

```java
// mapper/MemoMapper.java
@Mapper
public interface MemoMapper {
    // メモを新規作成
    int insert(Memo memo);
}
```

#### 3.2.4 Mapper XML（2waySQL）

```xml
<!-- mybatis/mapper/MemoMapper.xml -->
<insert id="insert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO memos (title, content)
    VALUES (#{title}, #{content})
</insert>
```

#### 3.2.5 DTO

```java
// dto/MemoDto.java
@Data
public class MemoDto {
    @NotBlank(message = "タイトルは必須です")
    @Size(max = 100, message = "タイトルは100文字以内")
    private String title;

    @Size(max = 5000, message = "内容は5000文字以内")
    private String content;
}
```

#### 3.2.6 Service

```java
// service/impl/MemoServiceImpl.java
@Service
public class MemoServiceImpl implements MemoService {

    private final MemoMapper memoMapper;

    @Override
    @Transactional
    public MemoDto create(MemoDto memoDto) {
        // DTOをEntityに変換
        Memo memo = new Memo();
        memo.setTitle(memoDto.getTitle());
        memo.setContent(memoDto.getContent());

        // データベースに保存
        memoMapper.insert(memo);

        // 保存したデータをDTOで返す
        return convertToDto(memo);
    }
}
```

#### 3.2.7 Controller

```java
// controller/MemoController.java
@Controller
@RequestMapping("/memos")
public class MemoController {

    private final MemoService memoService;

    // 新規作成フォームを表示
    @GetMapping("/new")
    public String newForm(Model model) {
        model.addAttribute("memoDto", new MemoDto());
        return "memos/new";  // templates/memos/new.html を表示
    }

    // フォームから送信されたデータを保存
    @PostMapping
    public String create(@Valid @ModelAttribute MemoDto memoDto,
                        BindingResult bindingResult,
                        RedirectAttributes redirectAttributes) {
        // バリデーションエラーがあればフォームを再表示
        if (bindingResult.hasErrors()) {
            return "memos/new";
        }

        // メモを保存
        memoService.create(memoDto);

        // 成功メッセージを設定
        redirectAttributes.addFlashAttribute("message", "メモを作成しました");

        // 一覧画面にリダイレクト
        return "redirect:/memos";
    }
}
```

#### 3.2.8 Thymeleafテンプレート

```html
<!-- templates/memos/new.html -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>新規メモ作成</title>
</head>
<body>
    <h1>新規メモ作成</h1>

    <!-- エラーメッセージ表示 -->
    <div th:if="${#fields.hasErrors('*')}" class="error">
        <p th:each="err : ${#fields.errors('*')}" th:text="${err}">Error</p>
    </div>

    <!-- 入力フォーム -->
    <form th:action="@{/memos}" th:object="${memoDto}" method="post">
        <div>
            <label>タイトル *</label>
            <input type="text" th:field="*{title}" required maxlength="100">
        </div>

        <div>
            <label>内容</label>
            <textarea th:field="*{content}" rows="10" maxlength="5000"></textarea>
        </div>

        <button type="submit">保存</button>
        <a th:href="@{/memos}">キャンセル</a>
    </form>
</body>
</html>
```

## 4. 3層アーキテクチャの実践

### 4.1 層ごとの責務

#### 4.1.1 プレゼンテーション層（Controller）

**責務**:
- ✅ HTTPリクエストを受け取る
- ✅ リクエストパラメータをDTOに変換
- ✅ バリデーションエラーの確認
- ✅ Serviceを呼び出す
- ✅ Viewに渡すデータを準備

**やってはいけないこと**:
- ❌ ビジネスロジックを書く（Serviceの仕事）
- ❌ SQLを実行する（Mapperの仕事）
- ❌ 直接Entityを扱う（DTOを使う）

**良い例**:
```java
@PostMapping
public String create(@Valid MemoDto memoDto, BindingResult result) {
    if (result.hasErrors()) {
        return "memos/new";
    }
    memoService.create(memoDto);  // Serviceに処理を委譲
    return "redirect:/memos";
}
```

**悪い例**:
```java
@PostMapping
public String create(@Valid MemoDto memoDto) {
    // ❌ ControllerでSQLを実行している
    String sql = "INSERT INTO memos (title, content) VALUES (?, ?)";
    jdbcTemplate.update(sql, memoDto.getTitle(), memoDto.getContent());
    return "redirect:/memos";
}
```

#### 4.1.2 ビジネスロジック層（Service）

**責務**:
- ✅ ビジネスルールの実装
- ✅ トランザクション管理（@Transactional）
- ✅ DTOとEntityの相互変換
- ✅ 複数のMapperを組み合わせた処理

**やってはいけないこと**:
- ❌ HTTPリクエスト/レスポンスを扱う
- ❌ 直接SQLを書く

**良い例**:
```java
@Service
@Transactional
public class MemoServiceImpl implements MemoService {

    private final MemoMapper memoMapper;

    @Override
    public MemoDto create(MemoDto memoDto) {
        // ビジネスルール: タイトルが空の場合はデフォルトタイトルを設定
        if (memoDto.getTitle() == null || memoDto.getTitle().isEmpty()) {
            memoDto.setTitle("無題のメモ");
        }

        Memo memo = convertToEntity(memoDto);
        memoMapper.insert(memo);
        return convertToDto(memo);
    }
}
```

#### 4.1.3 データアクセス層（Mapper）

**責務**:
- ✅ データベースへのCRUD操作
- ✅ SQLの実行
- ✅ 結果をEntityにマッピング

**やってはいけないこと**:
- ❌ ビジネスロジックを実装
- ❌ トランザクション制御（Serviceの仕事）

**良い例**:
```java
@Mapper
public interface MemoMapper {
    List<Memo> findAll();
    Memo findById(Long id);
    int insert(Memo memo);
    int update(Memo memo);
    int deleteById(Long id);
}
```

### 4.2 DTOとEntityの使い分け

#### 4.2.1 Entity（エンティティ）

**用途**: データベーステーブルの1行を表現

**特徴**:
- データベースのカラムと1対1で対応
- バリデーションアノテーションは**付けない**
- MyBatisで取得したデータを格納

```java
@Data
public class Memo {
    private Long id;
    private String title;
    private String content;
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

#### 4.2.2 DTO（データ転送オブジェクト）

**用途**: 画面とサービス層の間でデータ転送

**特徴**:
- バリデーションアノテーションを付ける
- 画面に必要なフィールドのみ含む
- Entityと完全に一致しなくてもOK

```java
@Data
public class MemoDto {
    private Long id;

    @NotBlank(message = "タイトルは必須です")
    @Size(max = 100, message = "100文字以内")
    private String title;

    @Size(max = 5000, message = "5000文字以内")
    private String content;

    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

#### 4.2.3 変換方法

**EntityからDTOへ**:
```java
private MemoDto convertToDto(Memo memo) {
    MemoDto dto = new MemoDto();
    dto.setId(memo.getId());
    dto.setTitle(memo.getTitle());
    dto.setContent(memo.getContent());
    dto.setCreatedAt(memo.getCreatedAt());
    dto.setUpdatedAt(memo.getUpdatedAt());
    return dto;
}
```

**DTOからEntityへ**:
```java
private Memo convertToEntity(MemoDto dto) {
    Memo memo = new Memo();
    memo.setTitle(dto.getTitle());
    memo.setContent(dto.getContent());
    // idやcreatedAtは設定しない（データベースが自動設定）
    return memo;
}
```

## 5. コーディング規約

### 5.1 命名規則

#### 5.1.1 クラス名

| 種類 | 命名規則 | 例 |
|------|---------|---|
| Controller | `名詞 + Controller` | `MemoController` |
| Service Interface | `名詞 + Service` | `MemoService` |
| Service Implementation | `名詞 + ServiceImpl` | `MemoServiceImpl` |
| Mapper | `名詞 + Mapper` | `MemoMapper` |
| Entity | `名詞` | `Memo` |
| DTO | `名詞 + Dto` | `MemoDto` |
| Exception | `説明 + Exception` | `MemoNotFoundException` |

#### 5.1.2 メソッド名

| 操作 | 命名規則 | 例 |
|------|---------|---|
| 取得（複数） | `findAll`, `findBy~` | `findAllOrderByCreatedAtDesc()` |
| 取得（単一） | `findById`, `getById` | `findById(Long id)` |
| 作成 | `create`, `save`, `insert` | `create(MemoDto dto)` |
| 更新 | `update` | `update(Long id, MemoDto dto)` |
| 削除 | `delete`, `deleteById` | `deleteById(Long id)` |
| 検索 | `searchBy~` | `searchByKeyword(String keyword)` |
| 存在確認 | `existsBy~` | `existsById(Long id)` |
| カウント | `count`, `countBy~` | `countByKeyword(String keyword)` |

#### 5.1.3 変数名

```java
// 良い例
List<Memo> memos = memoMapper.findAll();
MemoDto memoDto = new MemoDto();
String keyword = request.getParameter("keyword");

// 悪い例
List<Memo> m = memoMapper.findAll();        // 短すぎる
MemoDto data = new MemoDto();               // 抽象的すぎる
String s = request.getParameter("keyword"); // 意味不明
```

## 7. テストの書き方

### 7.1 テストの種類

| テスト種類 | 対象 | ツール |
|-----------|------|--------|
| 単体テスト | Mapper, Service, Controller | JUnit 5, Mockito |
| 統合テスト | 複数のクラスを組み合わせた動作 | Spring Boot Test |
| E2Eテスト | ブラウザ操作 | Selenium（高度） |

### 7.2 Mapperテスト

```java
@MybatisTest  // MyBatis用のテスト設定を自動で行う
class MemoMapperTest {

    @Autowired
    private MemoMapper memoMapper;
```

## 8. よくある問題と解決方法

### 8.1 コンパイルエラー

#### 問題: `Cannot resolve symbol 'Lombok'`

**原因**: Lombok プラグインがインストールされていない

**解決方法**:
```
1. IntelliJ IDEA → Settings → Plugins
2. "Lombok" を検索してインストール
3. 再起動
```

#### 問題: `Cannot resolve method 'xxx'`

**原因**: Mavenの依存関係がダウンロードされていない

**解決方法**:
```
IntelliJ IDEA の右サイドバー → Maven → Reload All Maven Projects
```

### 8.2 実行時エラー

#### 問題: `Port 8080 is already in use`

**原因**: 別のアプリケーションがポート8080を使用中

**解決方法**:
```yaml
# application.yml
server:
  port: 8081  # 別のポートに変更
```

#### 問題: `Connection refused` (PostgreSQL)

**原因**: PostgreSQLコンテナが起動していない

**解決方法**:
```bash
docker compose up -d
docker compose ps  # 起動確認
```

#### 問題: `Flyway migration failed`

**原因**: マイグレーションファイルにエラー、または既にテーブルが存在

**解決方法**:
```bash
# 状態確認
mvn flyway:info

# 修復
mvn flyway:repair

# 完全リセット（データ消える！）
mvn flyway:clean
mvn flyway:migrate
```

### 8.3 画面表示エラー

#### 問題: Thymeleafで `null` が表示される

**原因**: Controllerでモデルに値を設定していない

**解決方法**:
```java
@GetMapping
public String list(Model model) {
    model.addAttribute("memos", memoService.findAll());  // ← これを忘れずに！
    return "memos/list";
}
```

#### 問題: CSSが反映されない

**原因**: キャッシュが残っている

**解決方法**:
```
ブラウザで Ctrl+F5 (強制リロード)
```

### 9.1 基本的なGitフロー


### 9.2 コミットメッセージの書き方

#### 9.2.1 プレフィックス

| プレフィックス | 意味 | 例 |
|--------------|------|---|
| `feat:` | 新機能 | `feat: メモ検索機能を追加` |
| `fix:` | バグ修正 | `fix: 削除時のNullPointerExceptionを修正` |
| `docs:` | ドキュメント | `docs: READMEに環境構築手順を追加` |
| `refactor:` | リファクタリング | `refactor: Service層のコードを整理` |
| `test:` | テスト | `test: MemoServiceのテストを追加` |
| `chore:` | その他 | `chore: pom.xmlの依存関係を更新` |

#### 9.2.2 良いコミットメッセージ

```
✅ 良い例
feat: メモにカテゴリー機能を追加
fix: タイトル未入力時のバリデーションエラーを修正
docs: 環境構築手順書を追加

❌ 悪い例
update
修正
バグ直した
```

### 9.3 ブランチ戦略（シンプル版）

```bash
# 新機能開発用のブランチを作成
git checkout -b feature/add-category

# 開発してコミット
git add .
git commit -m "feat: カテゴリー機能を追加"

# メインブランチにマージ
git checkout main
git merge feature/add-category

# ブランチ削除
git branch -d feature/add-category
```
